## Overview

The Smart Voting Contract uses several configuration files to manage deployment settings, network configurations, and build options. This guide covers all customization options available.

## Foundry Configuration

The `foundry.toml` file contains Foundry-specific build and test configurations.

### Current Configuration

```toml foundry.toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]

fs_permissions = [
    { access = "read", path = "./broadcast" },
    { access = "read", path = "./out" }
]
```

### Configuration Options

#### Directory Structure

- `src`: Source directory for Solidity contracts (foundry.toml:2)
- `out`: Output directory for compiled artifacts (foundry.toml:3)
- `libs`: Dependencies directory (foundry.toml:4)

#### File System Permissions

Grants scripts read access to specific directories (foundry.toml:7-9):

- `./broadcast`: Deployment transaction records
- `./out`: Compiled contract artifacts

These permissions allow the interaction scripts to read deployment data and find recently deployed contracts.

### Common Customizations

Add these options to `foundry.toml` as needed:

<CodeGroup>
```toml Optimizer Settings
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
optimizer = true
optimizer_runs = 200

fs_permissions = [
    { access = "read", path = "./broadcast" },
    { access = "read", path = "./out" }
]
```

```toml Gas Reports
[profile.default]
src = "src"
out = "out"
libs = ["lib"]

    { access = "read", path = "./broadcast" },
```

## Network Configuration

The `HelperConfig.s.sol` script manages network-specific deployment parameters.

### Entry Fee Configuration

Modify the entry fee for different networks:

```solidity script/HelperConfig.s.sol
function getSepoliaConfig() public pure returns (NetworkConfig memory) {
    NetworkConfig memory sepoliaConfig = NetworkConfig({entryFee: 0.01 ether});
    return sepoliaConfig;
}

function getOrCreateAnvilConfig() public pure returns (NetworkConfig memory) {
    NetworkConfig memory anvilConfig = NetworkConfig({entryFee: 0.01 ether});
    return anvilConfig;
}
```

### Changing the Entry Fee

To set a different entry fee:

<Steps>
  <Step title="Edit HelperConfig.s.sol">
    Open `script/HelperConfig.s.sol` and modify the entry fee value:
    
    ```solidity
    function getSepoliaConfig() public pure returns (NetworkConfig memory) {
        NetworkConfig memory sepoliaConfig = NetworkConfig({entryFee: 0.05 ether}); // Changed from 0.01
        return sepoliaConfig;
    }
    ```
  </Step>

  <Step title="Rebuild contracts">
    Recompile to ensure changes are included:
    ```bash
    forge build
    ```
  </Step>

  <Step title="Deploy with new configuration">
    Deploy the contract with the updated entry fee:
    ```bash
    make deploy ARGS="--network sepolia"
    ```
  </Step>
</Steps>

<Warning>
  Changing the entry fee in HelperConfig only affects new deployments. Existing deployed contracts will continue using their original entry fee (set as immutable in MemberVote.sol:27).
</Warning>

### Adding New Networks

Add support for additional networks like Mainnet or Polygon:

```solidity script/HelperConfig.s.sol
constructor() {
    if (block.chainid == 11155111) {
        activeNetworkConfig = getSepoliaConfig();
    } else if (block.chainid == 1) {
        activeNetworkConfig = getMainnetConfig();
    } else if (block.chainid == 137) {
        activeNetworkConfig = getPolygonConfig();
    } else {
        activeNetworkConfig = getOrCreateAnvilConfig();
    }
}

function getMainnetConfig() public pure returns (NetworkConfig memory) {
    NetworkConfig memory mainnetConfig = NetworkConfig({entryFee: 0.001 ether});
    return mainnetConfig;
}

function getPolygonConfig() public pure returns (NetworkConfig memory) {
    NetworkConfig memory polygonConfig = NetworkConfig({entryFee: 1 ether}); // 1 MATIC
    return polygonConfig;
}
```

Common chain IDs:
- Ethereum Mainnet: `1`
- Sepolia: `11155111`
- Polygon: `137`
- Arbitrum: `42161`
- Optimism: `10`

## Environment Variables

Create a `.env` file for network-specific settings:

```bash .env
# Sepolia Configuration
SEPOLIA_RPC_URL=https://eth-sepolia.g.alchemy.com/v2/YOUR_API_KEY
PRIVATE_KEY=your_private_key_without_0x
ETHERSCAN_API_KEY=your_etherscan_api_key

# Voting Configuration
VOTE_OPTION=0  # 0 for Option A, 1 for Option B
```

### Required Variables for Sepolia

- `SEPOLIA_RPC_URL`: Alchemy or Infura RPC endpoint
- `PRIVATE_KEY`: Your wallet's private key (without 0x prefix)
- `ETHERSCAN_API_KEY`: For contract verification

### Optional Variables

- `VOTE_OPTION`: Default vote option (0 or 1). Defaults to 0 if not set (Interactions.s.sol:40)

<Warning>
  Never commit `.env` files to version control. Add `.env` to your `.gitignore` file.
</Warning>

## Contract Configuration

The MemberVote contract has several immutable configuration values set at deployment:

### Entry Fee

Set during deployment and cannot be changed (MemberVote.sol:27):

```solidity src/MemberVote.sol
uint256 private immutable i_entryFee;

constructor(uint256 entryFee) {
    i_owner = msg.sender;
    i_entryFee = entryFee;
    // ...
}
```

To use a different entry fee, deploy a new contract instance.

### Owner Address

Set to `msg.sender` at deployment and cannot be changed (MemberVote.sol:23):

```solidity src/MemberVote.sol
address private immutable i_owner;

constructor(uint256 entryFee) {
    i_owner = msg.sender;
    // ...
}
```

<Info>
  The contract does not include owner transfer functionality. The deployer remains the permanent owner.
</Info>

### Initial State

The contract starts in the Registering workflow station (MemberVote.sol:36):

```solidity src/MemberVote.sol
constructor(uint256 entryFee) {
    i_owner = msg.sender;
    i_entryFee = entryFee;
    s_electionId = 1;
    s_workFlowStation = WorkFlowStation.Registering;
}
```

## Deployment Script Configuration

The `DeployMemberVote` script (script/MemberVote.s.sol:8) is minimal and doesn't require configuration:

```solidity script/MemberVote.s.sol
contract DeployMemberVote is Script {
    function run() external returns (MemberVote, HelperConfig) {
        HelperConfig helperConfig = new HelperConfig();
        (uint256 entryFee) = helperConfig.activeNetworkConfig();
        vm.startBroadcast();
        MemberVote memberVote = new MemberVote(entryFee);
        vm.stopBroadcast();
        return (memberVote, helperConfig);
    }
}
```

All configuration is handled through HelperConfig, making the deployment script network-agnostic.

## Configuration Best Practices

1. **Never hardcode secrets**: Use environment variables for private keys and API keys
2. **Use different entry fees per network**: Lower fees for testnets, higher for mainnet
3. **Enable verification**: Always verify on public networks for transparency
4. **Version control**: Commit `foundry.toml` and `Makefile`, never commit `.env`
5. **Document changes**: Comment why you changed default configurations
