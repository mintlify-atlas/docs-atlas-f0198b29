This guide will help you deploy and interact with the MemberVote contract locally. You'll learn how to set up the project, deploy the contract, start a vote, and cast votes.

## Prerequisites

Before you begin, ensure you have [Foundry](https://book.getfoundry.sh/getting-started/installation) installed:

```bash
curl -L https://foundry.paradigm.xyz | bash
foundryup
```

## Installation

<Steps>
  <Step title="Clone and navigate to the project">
    ```bash
    git clone <your-repo-url>
    cd smart-voting-contract
    ```
  </Step>

  <Step title="Install dependencies">
    Install Foundry dependencies including foundry-devops and forge-std:
    
    ```bash
    make install
    ```
  </Step>

  <Step title="Build the project">
    Compile the smart contracts:
    
    ```bash
    make build
    ```
  </Step>

  <Step title="Run tests">
    Verify everything works correctly:
    
    ```bash
    make test
    ```
  </Step>
</Steps>

## Local Deployment

<Steps>
  <Step title="Start local Anvil node">
    Open a terminal and start a local Ethereum node:
    
    ```bash
    make anvil
    ```
    
    This starts Anvil with a 1-second block time and deterministic test accounts.
  </Step>

  <Step title="Deploy the contract">
    In a new terminal, deploy the MemberVote contract:
    
    ```bash
    make deploy
    ```
    
    This deploys the contract to your local Anvil node using the default deployment key.
  </Step>

  <Step title="Start the voting process">
    Initialize voting by calling the `startVote` function:
    
    ```bash
    make start-vote
    ```
    
    This changes the contract workflow from `Registering` to `Voting` state, allowing members to cast votes.
  </Step>
</Steps>

## Casting Votes

Once voting is active, members can cast their votes for either Option A (0) or Option B (1).

### Vote for Option A

Set the vote option environment variable and cast your vote:

```bash
export VOTE_OPTION=0
make vote
```

### Vote for Option B

```bash
export VOTE_OPTION=1
make vote
```

### How voting works

The `vote` function in MemberVote.sol (src/MemberVote.sol:76) accepts an option parameter and entry fee:

```solidity
function vote(uint256 option) public payable {
    address voter = msg.sender;
    if (msg.value < i_entryFee) {
        revert MemberVote__InvalidEntryFee();
    }
    if (s_workFlowStation != WorkFlowStation.Voting) {
        revert MemberVote__WrongWorkflowStation();
    }
    if (s_addressToVoted[voter] == s_electionId) {
        revert MemberVote__AlreadyVoted();
    }
    if (option != 0 && option != 1) {
        revert MemberVote__InvalidOption();
    }
    // Records vote and increments count
}
```

<Info>
  **Entry Fee**: Each vote requires an entry fee (configured during deployment). The contract validates the fee amount before accepting the vote.
</Info>

## Resetting Elections

When you're ready to start a new election round, reset the votes:

```bash
make reset-vote
```

This increments the election ID and transitions the workflow to the `Resetting` state, preparing for a new voting cycle.

## Testnet Deployment (Optional)

To deploy to Sepolia testnet, you'll need to configure your environment:

<Steps>
  <Step title="Set up environment variables">
    Create a `.env` file in the project root:
    
    ```bash
    SEPOLIA_RPC_URL=your_sepolia_rpc_url
    PRIVATE_KEY=your_private_key
    ETHERSCAN_API_KEY=your_etherscan_api_key
    ```
    
    <Warning>
      Never commit your `.env` file to version control. Keep your private keys secure.
    </Warning>
  </Step>

  <Step title="Deploy to Sepolia">
    ```bash
    make deploy ARGS="--network sepolia"
    ```
    
    This will deploy the contract and verify it on Etherscan automatically.
  </Step>

  <Step title="Interact with testnet contract">
    Use the same commands with the network argument:
    
    ```bash
    make start-vote ARGS="--network sepolia"
    make vote ARGS="--network sepolia"
    make reset-vote ARGS="--network sepolia"
    ```
  </Step>
</Steps>
