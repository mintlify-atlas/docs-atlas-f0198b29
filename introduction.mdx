## What is Smart Voting Contract?

Smart Voting Contract is a Solidity-based voting system designed for on-chain governance, polls, and decision-making. It implements a binary choice voting mechanism (Option A vs Option B) with configurable entry fees and owner-controlled election cycles.

The contract is built by [Pragyat Nikunj](https://github.com/pragyat) and leverages Foundry's development toolkit for testing, deployment, and interaction scripts.

- **Smart contract learners** studying real-world Solidity patterns and Foundry workflows

<Note>
  The contract implements a binary voting system (Option A vs Option B). For multi-option voting or more complex governance mechanisms, you'll need to extend the base contract.
</Note>

## How it works

The voting process follows a simple three-stage workflow:

1. **Registering**: Initial state where the contract owner prepares for the next election
2. **Voting**: Active voting period where members can cast their votes by paying the entry fee
3. **Resetting**: Post-voting state where the owner resets the contract for the next election cycle

Each voter must pay a minimum entry fee (default: 0.01 ETH) to cast their vote. The contract prevents double voting by tracking each address against the current election ID.

```solidity
// Example: Voting for Option A
function vote(uint256 option) public payable {
    // Checks entry fee
    if (msg.value < i_entryFee) {
        revert MemberVote__InvalidEntryFee();
    }
    // Validates workflow station is Voting
    if (s_workFlowStation != WorkFlowStation.Voting) {
        revert MemberVote__WrongWorkflowStation();
    }
    // Prevents double voting
    if (s_addressToVoted[voter] == s_electionId) {
        revert MemberVote__AlreadyVoted();
    }
    // Records vote
    s_addressToVoted[voter] = s_electionId;
    voters.push(voter);
}
```

<Warning>
  Always ensure the contract is in the correct workflow station before attempting to vote. Calls made during the wrong phase will revert.
</Warning>
