## Test Patterns Used

### Arrange-Act-Assert

Tests follow the AAA pattern:

```solidity
function testIfVoterCanVote() public {
    // Arrange
    (uint256 entryFee) = helperConfig.activeNetworkConfig();
    vm.prank(msg.sender);
    memberVote.startVote();
    
    // Act
    vm.prank(USER1);
    memberVote.vote{value: entryFee}(OPTION_A);
    
    // Assert
    assertEq(memberVote.getOptionAVotes(), STARTING_OPTION_A_VOTES + 1);
}
```

### Negative Testing

Tests verify both success and failure cases:

```solidity
// Positive test
function testIfOwnerCanStartVoting() public {
    vm.prank(msg.sender);
    memberVote.startVote();
    assertEq(uint256(memberVote.getWorkflowStation()), 1);
}

// Negative test
function testIfVoterCannotStartVoting() public {
    vm.prank(USER1);
    vm.expectRevert(MemberVote.MemberVote__NotOwner.selector);
    memberVote.startVote();
}
```

### State Management

Each test gets a fresh contract instance via `setUp()`, ensuring test isolation.
